## Release Note

The first publish overlapped with the tag-triggered GitHub Actions release
workflow. The release had all expected assets, but the downloaded darwin arm64
binary initially printed `dev` because `.github/workflows/release.yml` built
without `-X main.helperVersion=$GITHUB_REF_NAME`.

Fix:

- updated the release workflow on `main` to embed `main.helperVersion`;
- rebuilt local assets with `scripts/build-release-assets.sh v0.3.10
  dist/v0.3.10`;
- republished/replaced all `v0.3.10` assets;
- redownloaded darwin arm64 and confirmed `version` prints `v0.3.10`.
